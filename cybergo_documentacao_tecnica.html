Arquitetura do Sistema

Visão Geral

O CyberGo utiliza o framework Django, seguindo o padrão MTV:

  • Model: Camada de dados
  • Template: Camada de apresentação
  • View: Camada de lógica de negócios

Tecnologias Utilizadas

  • Python 3.8+ com Django 5.1.x
  • SQLite (desenvolvimento)
  • Bootstrap 5, HTML5, CSS3, JavaScript
  • django-crispy-forms, django-import-export...

Estrutura de Diretórios

Projeto Operacional/
config/ # Configurações Django
core/ # Aplicativo principal
escolas/ # Gestão de escolas

Middlewares e Extensões

Middleware Personalizado

class PerfilAcessoMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response
        
    def __call__(self, request):
        if request.user.is_authenticated:
            perfil = request.user.perfil
            request.perfil_acesso = perfil.tipo
            
            # Verifica restrições de acesso
            if perfil.is_restricted and not self.is_allowed_path(request.path):
                return redirect('acesso_negado')
                
        return self.get_response(request)

Implementa verificação de acesso baseada no perfil do usuário.

@register.filter
def format_currency(value):
    """Formata um valor para moeda brasileira"""
    if value is None:
        return "R$ 0,00"
    return f"R$ {value:.2f}".replace('.', ',')
@register.simple_tag
def is_menu_active(request, pattern):
    """Verifica se um item de menu está ativo"""

        return "active"

def aprovar_pedidos(modeladmin, request, queryset):
    for pedido in queryset:
        if pedido.status == 'PENDENTE':
            pedido.aprovar(usuario=request.user)
    
    modeladmin.message_user(
        request,
        f"{queryset.count()} pedidos aprovados com sucesso!",

def listar_pedidos(request):
    # Carrega escola e itens em uma única consulta
    pedidos = Pedido.objects.select_related('escola')\
              .prefetch_related('itens__produto')\
              .filter(status='APROVADO')\
              .order_by('-data_solicitacao')

    Fluxos Técnicos Detalhados

    Fluxo de Pedido

    1. Criação: Escola inicia pedido (status: Rascunho)
    2. Envio: Escola finaliza pedido (status: Pendente)
    3. Aprovação: Supervisor avalia pedido
      • Se aprovado: status → Aprovado
      • Se rejeitado: status → Rejeitado
    4. Processamento: Operador prepara pedido (status: Em Processamento)
    5. Envio: Logística envia pedido (status: Enviado)
    6. Entrega: Escola confirma recebimento (status: Entregue)

    Fluxo de Cadastro de Escola

    1. Registro inicial: Dados básicos da escola
    2. Atribuição de supervisor: Um supervisor é designado
    3. Definição de orçamento: Budget é configurado
    4. Credenciais: Sistema gera credenciais de acesso
    5. Notificação: E-mail enviado com instruções
    6. Primeiro acesso: Escola define senha permanente

    Todos os fluxos incorporam hooks para auditoria, validações em múltiplas camadas, e sistema de recuperação em caso de falhas.

    class PedidoExcedeLimiteError(Exception):
    """Erro quando pedido excede limite orçamentário"""
    pass

def criar_pedido(request):
    try:
        # Lógica de criação de pedido
        if valor_total > escola.budget_disponivel:
            raise PedidoExcedeLimiteError(
                "Pedido excede o limite orçamentário")
        
        # Continua processamento normal
        return redirect('pedido_sucesso')
        
    except PedidoExcedeLimiteError as e:
        messages.error(request, str(e))
        logger.warning(
            f"Tentativa acima do limite: {request.user}")
        return render(request, 'pedido_form.html', 
                     {'form': form, 'erro': e})

    class PedidoTestCase(TestCase):
    def setUp(self):
        # Configuração inicial para os testes
        self.escola = Escola.objects.create(
            nome="Escola Teste", budget=1000.00)
        self.produto = Produto.objects.create(
            nome="Produto Teste", valor_unitario=100.00)
    
    def test_criar_pedido_valido(self):
        """Testa a criação de um pedido válido"""
        pedido = Pedido.objects.create(
            escola=self.escola, status="RASCUNHO")
            
        ItemPedido.objects.create(
            pedido=pedido,
            produto=self.produto,
            quantidade=5)
            
        self.assertEqual(pedido.valor_total(), 500.00)
        self.assertTrue(pedido.validar_orcamento())
        pedido.atualizar_status("PENDENTE")
        self.assertEqual(pedido.status, "PENDENTE")
